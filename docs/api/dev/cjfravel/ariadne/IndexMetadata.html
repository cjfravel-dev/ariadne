IndexMetadata 

Companion object IndexMetadata

case class IndexMetadata(format: String, schema: String, indexes: List[String], computed_indexes: Map[String, String], exploded_field_indexes: List[ExplodedFieldMapping], bloom_indexes: List[BloomIndexConfig], temporal_indexes: List[TemporalIndexConfig], read_options: Map[String, String], range_indexes: List[RangeIndexConfig], auto_bloom_indexes: List[String], total_indexed_file_size: Long, batches_since_compact: Integer, metadata_version: Integer, storage_format_version: Integer) extends Product with Serializable

Metadata container for Ariadne index configuration and state.

This case class stores all metadata required to manage an Ariadne index, including schema information, indexed columns, and format details. It supports multiple versions for backward compatibility.

format

The file format of the indexed data (e.g., "parquet", "csv", "json")

schema

The JSON representation of the DataFrame schema

indexes

List of regular column names that are indexed

computed_indexes

Map of computed index aliases to their SQL expressions

exploded_field_indexes

List of exploded field mappings for nested data structures

bloom_indexes

List of bloom filter index configurations for probabilistic indexing

temporal_indexes

List of temporal index configurations for version-aware deduplication

read_options

Map of read options for format-specific configuration (e.g., "multiLine" -> "true" for JSON)

total_indexed_file_size

Total size in bytes of all indexed files, or -1 if not yet computed (nullable for Gson compatibility)

batches_since_compact

Number of update batches processed since the last auto-compaction, persisted across Spark jobs so that autoCompactThreshold works correctly even when updates happen in separate runs (v10+, nullable for Gson)

metadata_version

Explicit additive metadata schema version, or null for indexes created before explicit versioning

storage_format_version

Explicit whole-index physical storage version, or null for unversioned alpha37+ indexes

Note

The field names use underscore notation to match JSON serialization format. All fields use var and Java collections (util.List, util.Map) because Gson requires mutable fields for deserialization. Do not refactor to val or Scala collections without updating the Gson serialization in IndexMetadata.apply and IndexMetadataOperations.
