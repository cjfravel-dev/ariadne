Find the thread through your data lake.

Ariadne builds lightweight file-level indexes over Parquet, CSV, and JSON files so Spark joins read only the files they actually need to.

Get Started View on GitHub

Why Ariadne?

If you're joining a DataFrame against a large collection of files, Spark normally has to read all of them. Ariadne builds lightweight indexes over those files so that only the relevant ones are loaded — skipping files that can't contain matching rows.

Like Ariadne from Greek mythology, this library helps you navigate your data labyrinth. And just like Ariadne, I hope you'll one day betray it — like Theseus — and move on to something better, such as Apache Iceberg or Delta Lake proper. But in the meantime, if your data lake is more of a data swamp, Ariadne can help.

What you get

⊞

Six index types

Regular, bloom, range, temporal, computed, and exploded-field indexes — combine them with AND semantics on multi-column joins.

⚡

Drop-in joins

Use Ariadne indexes in normal Spark joins: df.join(index, …). Pruning happens automatically — your existing query plans stay readable.

Join patterns
𝚫

Delta-backed

Index metadata and data live in Delta Lake tables. ACID, time-travel, compactable, and durable on any Hadoop-accessible filesystem.

Spark SQL catalog

Register the Ariadne catalog and query indexes as tables: SELECT … FROM ariadne.orders WHERE … — Catalyst prunes files for you.

SQL catalog
⏱

Temporal dedup

Same entity in multiple files over time? A temporal index keeps only the latest row per key during the join — no manual window functions.

Temporal indexes
⌖

Production-ready

File locking, stale-lock auto-heal, batched updates, auto-compaction, and pruning metrics built in. No surprises in long-running jobs.

A quick look

import dev.cjfravel.ariadne.Index
import dev.cjfravel.ariadne.Index._  // for df.join(index, ...) implicit

spark.conf.set("spark.ariadne.storagePath",
  s"abfss://$container@$account.dfs.core.windows.net/ariadne")

// 1. Define the index
val index = Index("orders", orderSchema, "parquet")
index.addIndex("customer_id")
index.addFile(orderFiles: _*)
index.update

// 2. Use it in a join — Ariadne only loads files that match
val result = customerDf.join(index, Seq("customer_id"), "inner")
